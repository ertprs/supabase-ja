Supabaseのデータベースは、デフォルトでUTCタイムゾーンに設定されています。これは、「私のデータベースの中のものはすべてUTC時間である」というメンタルモデルを採用すると、タイムゾーン間の差を計算するのが非常に簡単になるからです。


### タイムゾーンの変更

<!-- textlint-disable -->

<Tabs
  defaultValue="sql"
  groupId="reference/postgres"
  values={[{ label: 'SQL', value: 'sql' }]}>

<TabItem value="sql">

<!-- textlint-enable -->

```sql
alter database postgres 
set timezone to 'America/New_York';
```


</TabItem>

</Tabs>

### タイムゾーンの全リスト

データベースでサポートされているタイムゾーンの完全なリストを取得します。これにより、以下の列が返されます。

- `name`: タイムゾーン名
- `abbrev`：タイムゾーンの略語
- `utc_offset`：UTCからのオフセット。UTCからのオフセット（正の値はグリニッジの東を意味します）
- `is_dst`:現在サマータイムが適用されている場合はtrue

<!-- textlint-disable -->

<Tabs
  defaultValue="sql"
  groupId="reference/postgres"
  values={[{ label: 'SQL', value: 'sql' }]}>

<TabItem value="sql">

<!-- textlint-enable -->

```sql
select name, abbrev, utc_offset, is_dst
from pg_timezone_names() 
order by name;
```


</TabItem>

</Tabs>

### 特定のタイムゾーンの検索

特定のタイムゾーンを検索するには、`ilike`（大文字と小文字を区別しない検索）を使用します。

<!-- textlint-disable -->

<Tabs
  defaultValue="sql"
  groupId="reference/postgres"
  values={[{ label: 'SQL', value: 'sql' }]}>

<TabItem value="sql">

<!-- textlint-enable -->

```sql
select * 
from pg_timezone_names() 
where name ilike '%york%';
```


</TabItem>

</Tabs>
